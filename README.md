# 应急帐篷调拨墙

面向物资仓库、安置点管理员和指挥部三类视角的交互式Web前端系统。

## 功能特性

### 🎯 三类视角
- **物资仓库视角**: 批次库存管理、出库分配记录
- **安置点管理员视角**: 安置点状态监控、接收分配记录
- **指挥部视角**: 全局库存概览、安置点调度、全量分配记录、快捷操作

### 📦 核心功能
1. **帐篷批次管理**: 查看各规格帐篷库存状态（可用/锁定/已发运）
2. **批次拆分分配**: 从批次拆分帐篷分配到多个安置点
3. **缺口计算**: 实时计算各安置点人员缺口
4. **容量校验**: 目标点容量不足时显示超出人数和建议拆分方案
5. **运输锁定**: 分配后锁定库存，发运后扣减
6. **撤销分配**: 支持撤销未发运的锁定状态分配
7. **优先级抢占**: 高优先级安置点可抢占低优先级的锁定库存
8. **指挥清单导出**: 导出CSV格式的指挥调度清单

### 📋 业务规则
- ✅ 已发运批次不能二次拆分
- ✅ 运输中分配不能改派
- ✅ 容量不足时显示超出人数和建议拆分方案
- ✅ 高优先级抢占库存时回写其他点缺口
- ✅ 仅可撤销锁定状态（未发运）的分配

## 快速开始

### 方式一：本地静态预览
```bash
# 启动本地HTTP服务器
python3 -m http.server 8080

# 浏览器访问
open http://localhost:8080
```

### 方式二：Docker容器运行
```bash
# 构建镜像
npm run docker:build

# 运行容器
npm run docker:run

# 浏览器访问
open http://localhost:8080
```

### 方式三：直接打开
直接在浏览器中打开 `index.html` 文件即可使用（部分浏览器可能限制本地文件功能）。

## 数据模型

### 帐篷规格 (TentSpecs)
- 5种规格：12人标准、24人大型、6人家庭、医疗专用、指挥通讯

### 批次库存 (TentBatches)
- 5个批次，包含可用、锁定、已发运数量
- 支持按仓库存储

### 安置点 (Shelters)
- 5个安置点，包含容量、当前人数、优先级
- 实时计算缺口人数

### 分配记录 (AllocationRecords)
- 6条历史分配记录
- 状态：锁定/运输中/已发运/已撤销/已被抢占

## 操作指南

### 指挥部视角操作
1. **新建分配**: 点击「新建分配」选择批次、安置点和数量
2. **批量发运**: 点击「批量发运」一键发运所有锁定状态分配
3. **高优先级抢占**: 点击「高优先级抢占」从低优先级分配中抢占库存
4. **导出清单**: 点击「导出指挥清单」下载CSV文件

### 分配状态流转
```
锁定 (locked) → 运输中 (in_transit) → 已送达 (delivered)
    ↓
  已撤销 (cancelled) / 已被抢占 (preempted)
```
